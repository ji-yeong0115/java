Hello, JSP!

JSP(Java Server Page)란, Java 코드가 포함된 HTML 문서를 말하며
Servlet에서 복잡하게 작성하였던 프레젠테이션 로직을
별도의 JSP로 분리해 작성할 수 있다.

JSP는 요청에 대한 응답 시 Servlet으로 변환되는데
개발자가 직접 코딩한 Servlet보다 더욱 최적화된 형태의 Servlet으로 변환을 시켜줌

JSP - 1

스크립팅 원소(Scripting Element)
JSP 페이지에서 자바 코드를 직접 기술할 수 있게 해주는 기능
1) 선언문 <%! 자바 코드 %>
멤버 변수, 멤버 메소드를 선언하기 위해 사용됨
2) 스크립틀릿(scriptlet) : <% 자바 코드 %>
JSP에서 자바 코드를 작성하기 위해 사용됨(선언, 로직구현, 출력)
3) 출력식(표현식 : expression) : <%= 자바코드 %>
자바 변수에 저장된 값 또는 메소드 반환 값 등을 화면에 출력할 때 사용됨

jsp - 2

JSP의 맨 윗 부분에는 JSP 파일의 속성을 기술하는 page 지시어가 존재함.
language는 사용할 스크립트 언어 유형을 지정
contentType은 웹 브라우저가 받아 볼 페이지의 형식, 인코딩 방식 지정
pageEncoding은 JSP 파일에 기록된 자바코드의 인코딩 방식을 지정
import는 자바의 import와 같은 의미
errorPage / isErrorPage 는 오류페이지 관리

JSP - 3

정적 include로 페이지 안으로 include한 페이지의 소스가 단순 텍스트로 그대로 복사되어 포함된다.
include한 페이지 소스 내부에 변수가 선언되어 있으면 베이스 페이지의 변수와 충돌이 날 수 있으므로 유의해야한다.
include 페이지 안의 import는 같이 공유된다.
주로 페이지를 조각내어 필요한 부분에 삽입할 때 사용한다.
